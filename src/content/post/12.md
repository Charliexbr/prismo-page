## 39. 故障排查速查表

### 39.1 常见问题快速诊断

#### 39.1.1 连接问题

**问题：无法连接到VPN**
```bash
# 快速诊断脚本
#!/bin/bash

echo "=== 快速诊断 ==="

# 1. 检查服务状态
echo "[1] 服务状态:"
systemctl is-active wg-quick@wg0 && echo "  ✓ 运行中" || echo "  ✗ 已停止"

# 2. 检查接口
echo "[2] 接口状态:"
ip link show wg0 &>/dev/null && echo "  ✓ 存在" || echo "  ✗ 不存在"

# 3. 检查端口
echo "[3] 端口监听:"
ss -ulnp | grep :51820 &>/dev/null && echo "  ✓ 正在监听" || echo "  ✗ 未监听"

# 4. 检查防火墙
echo "[4] 防火墙:"
iptables -L INPUT -n | grep 51820 &>/dev/null && echo "  ✓ 已允许" || echo "  ⚠ 可能被阻止"

# 5. 检查配置
echo "[5] 配置文件:"
wg-quick strip wg0 &>/dev/null && echo "  ✓ 语法正确" || echo "  ✗ 语法错误"

# 6. 检查连通性
echo "[6] 网络连通性:"
ping -c 1 -W 2 8.8.8.8 &>/dev/null && echo "  ✓ 互联网可达" || echo "  ✗ 网络不通"
```

**解决方案速查：**

| 症状 | 可能原因 | 快速修复 |
|------|----------|----------|
| 服务未运行 | systemd问题 | `systemctl start wg-quick@wg0` |
| 端口未监听 | 配置错误 | 检查`wg0.conf` |
| 防火墙阻止 | 规则缺失 | `ufw allow 51820/udp` |
| 握手失败 | 密钥错误 | 验证公私钥匹配 |
| 连接后无流量 | 路由/NAT问题 | 检查IP转发和iptables |

#### 39.1.2 性能问题

**问题：速度慢或延迟高**
```bash
# 性能诊断脚本
#!/bin/bash

echo "=== 性能诊断 ==="

# 1. CPU使用率
echo "[1] CPU使用:"
top -bn1 | grep "Cpu(s)" | awk '{print "  ", $0}'

# 2. 内存使用
echo "[2] 内存使用:"
free -h | grep Mem | awk '{printf "  使用: %s/%s (%.1f%%)\n", $3, $2, ($3/$2)*100}'

# 3. 网络统计
echo "[3] 网络错误:"
ip -s link show wg0 | grep -E "RX:|TX:|errors"

# 4. MTU检查
echo "[4] MTU设置:"
ip link show wg0 | grep -o "mtu [0-9]*"

# 5. 延迟测试
echo "[5] VPN延迟:"
ping -c 5 10.8.0.1 2>/dev/null | grep "avg" || echo "  测试失败"

# 6. TCP拥塞控制
echo "[6] 拥塞控制:"
sysctl net.ipv4.tcp_congestion_control
```

**优化建议速查：**

| 问题 | 检查项 | 优化方案 |
|------|--------|----------|
| 高延迟 | MTU、CPU | 降低MTU、启用BBR |
| 低吞吐量 | 缓冲区、队列 | 增大缓冲区、调整队列长度 |
| 丢包 | 网络质量 | 调整保活间隔、检查物理链路 |
| CPU高 | 加密、中断 | 启用硬件加速、优化中断分布 |

### 39.3 配置模板

#### 39.3.1 服务器配置模板

**基础配置：**
```ini
[Interface]
# 服务器私钥
PrivateKey = <SERVER_PRIVATE_KEY>

# 服务器虚拟IP地址
Address = 10.8.0.1/24

# 监听端口
ListenPort = 51820

# MTU设置（可选）
MTU = 1420

# 启动后执行的命令
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT; iptables -t nat -A POSTROUTING -o pnet0 -j MASQUERADE
PostDown = iptables -D FORWARD -i wg0 -j ACCEPT; iptables -t nat -D POSTROUTING -o pnet0 -j MASQUERADE

# 客户端1
[Peer]
PublicKey = <CLIENT1_PUBLIC_KEY>
AllowedIPs = 10.8.0.2/32

# 客户端2
[Peer]
PublicKey = <CLIENT2_PUBLIC_KEY>
AllowedIPs = 10.8.0.3/32
```

**高级配置（含PSK）：**
```ini
[Interface]
PrivateKey = <SERVER_PRIVATE_KEY>
Address = 10.8.0.1/24
ListenPort = 51820
MTU = 1420

# 高级网络配置
PostUp = iptables -A FORWARD -i wg0 -j ACCEPT
PostUp = iptables -A FORWARD -o wg0 -j ACCEPT
PostUp = iptables -t nat -A POSTROUTING -o pnet0 -j MASQUERADE
PostUp = iptables -t mangle -A FORWARD -i wg0 -p tcp --tcp-flags SYN,RST SYN -j TCPMSS --clamp-mss-to-pmtu

PostDown = iptables -D FORWARD -i wg0 -j ACCEPT
PostDown = iptables -D FORWARD -o wg0 -j ACCEPT
PostDown = iptables -t nat -D POSTROUTING -o pnet0 -j MASQUERADE
PostDown = iptables -t mangle -D FORWARD -i wg0 -p tcp --tcp-flags SYN,RST SYN -j TCPMSS --clamp-mss-to-pmtu

[Peer]
PublicKey = <CLIENT_PUBLIC_KEY>
PresharedKey = <PRESHARED_KEY>
AllowedIPs = 10.8.0.2/32
```

#### 39.3.2 客户端配置模板

**分流模式（仅VPN网段）：**
```ini
[Interface]
PrivateKey = <CLIENT_PRIVATE_KEY>
Address = 10.8.0.2/24
DNS = 223.5.5.5

[Peer]
PublicKey = <SERVER_PUBLIC_KEY>
Endpoint = 192.168.1.66:51820
AllowedIPs = 10.8.0.0/24
PersistentKeepalive = 25
```

**全流量模式：**
```ini
[Interface]
PrivateKey = <CLIENT_PRIVATE_KEY>
Address = 10.8.0.2/24
DNS = 223.5.5.5, 8.8.8.8

[Peer]
PublicKey = <SERVER_PUBLIC_KEY>
Endpoint = 192.168.1.66:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
```

**多网段访问：**
```ini
[Interface]
PrivateKey = <CLIENT_PRIVATE_KEY>
Address = 10.8.0.2/24
DNS = 192.168.1.1

[Peer]
PublicKey = <SERVER_PUBLIC_KEY>
Endpoint = 192.168.1.66:51820
# VPN网段 + 内网网段
AllowedIPs = 10.8.0.0/24, 192.168.1.0/24, 172.16.0.0/16
PersistentKeepalive = 25
```

## 41. 术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| VPN | Virtual Private Network | 虚拟专用网络 |
| MTU | Maximum Transmission Unit | 最大传输单元 |
| NAT | Network Address Translation | 网络地址转换 |
| PSK | Pre-Shared Key | 预共享密钥 |
| BBR | Bottleneck Bandwidth and RTT | 拥塞控制算法 |
| RPS | Receive Packet Steering | 接收包控制 |
| RSS | Receive Side Scaling | 接收端扩展 |
| TSO | TCP Segmentation Offload | TCP分段卸载 |
| GSO | Generic Segmentation Offload | 通用分段卸载 |
| GRO | Generic Receive Offload | 通用接收卸载 |
| AEAD | Authenticated Encryption with Associated Data | 关联数据认证加密 |
| HKDF | HMAC-based Key Derivation Function | 基于HMAC的密钥派生函数 |
| Handshake | 握手 | 建立安全连接的过程 |
| Peer | 对等节点 | WireGuard中的连接端点 |
| AllowedIPs | 允许的IP | 定义路由和访问控制的IP范围 |
| Endpoint | 端点 | 对等节点的网络地址 |
